# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Running the demo

All commands must be run from inside `subsuelo/` (the inner project root):

```bash
cd subsuelo
pip install -r requirements.txt
python build.py               # reproducible build of ALL regions + web assets (the canonical command)
python run_region.py spain    # one region only: peninsular Spain (IGME) — raster + hotspots + Catastro drill
python run_region.py europe   # one region only: Variscan Europe (EGDI, 5 km) — raster + hotspots + border-aware drill
python run_demo.py            # synthetic data
python run_live.py            # Cáceres window: parcels-first + concessions friction
python run_ui.py --no-run     # serve the interactive Leaflet UI at http://127.0.0.1:8000
```

**Deploy** (the React app is a static site — bundle + pre-built data, no server): `cd webapp && ./deploy/build-static.sh` → self-contained `webapp/dist/` (js/css/html + a real copy of `out/web/regions`, symlink dereferenced; vite `base:'./'` so it works at a domain root or subpath). Drop `dist/` on any static host (Netlify/Vercel/Cloudflare Pages/GitHub Pages/S3 — see `webapp/DEPLOY.md`). Scheduled refresh: `DEPLOY_CMD="…" ./deploy/refresh.sh` runs `SUBSUELO_REFRESH=1 python build.py` → rebuild bundle → deploy.

## Reproducible builds / automatic updates

`build.py` is the single command that regenerates the whole page — it fetches (or replays from cache) every data source, runs the deterministic transforms, and writes each region's web assets + a manifest, in one shot. Because the transforms carry no randomness, **a build is reproducible from its raw-download cache** (verified: an offline rebuild reproduces every artifact byte-for-byte).

- **`subsuelo/ingest/net.py`** — every live fetch goes through `net.http_get(url, params, tag=…)`, a disk cache (`out/cache/`, keyed by url+sorted-params) that also appends to a `PROVENANCE` log (url, bytes, cache-hit, UTC). Atomic writes; a failed refresh falls back to the last cached copy (logged `stale`) so one flaky endpoint doesn't sink the build. Modes via env: `SUBSUELO_REFRESH=1` (refetch all + overwrite cache — the **scheduled-update** command), `SUBSUELO_OFFLINE=1` (cache only, no network — CI / source outage), `SUBSUELO_NO_CACHE=1`, `SUBSUELO_CACHE=<dir>`. (NE land/countries keep their own persistent geojson cache; idealista POST is unused.)
- **`build.py`** — `python build.py [regions…]` (default all). Per region: resets provenance → `pipeline.run` (returns a source-count summary) → `export.export` → writes `regions/<key>/provenance.json` and stamps `built_at` + `sources` into that region's `meta.json`. Regions are isolated (one failing is recorded and skipped, leaving its old assets intact); writes `out/web/build.json` (UTC time, mode, per-region status + counts) and exits non-zero if any region failed. For a cron/CI auto-update: `SUBSUELO_REFRESH=1 python build.py`.
- Catastro ATOM feeds mix encodings (national office index is **latin-1**, per-province feeds are **utf-8**) — `live._decode_atom` tries utf-8 then falls back to latin-1, so ñ-provinces (Coruña, Logroño) resolve correctly.

## Modular region engine (add a new area in ~10 lines)

The scalable core is a **Region provider** pattern:
- `subsuelo/regions.py` — the `Region` dataclass + `REGIONS` registry. Each area declares its grid (CRS, cell size, lon/lat window), its data-source callables (`granite`/`faults`/`occurrences`, each `bbox_4326 -> GeoDataFrame in region CRS`), WofE thresholds, and — where an open cadastre exists — `parcels_provider`/`concessions_provider`. Adding a country = one `Region(...)` entry.
- `subsuelo/pipeline.py` — the region-agnostic engine `run(region)`: fetch evidence → rasterize → WofE (combined + per-metal) → hotspots (density-weighted; sharpened to real clusters) → optional parcel drill (top 8, on demand in the UI) → `./out` artifacts. `run_national.py`/`run_europe.py`/`run_region.py` are thin wrappers; `run_hotspots.py` is folded into the `spain` region's drill.
- `subsuelo/ingest/live.py` — pluggable data-source functions (IGME, EGDI, Catastro, MITECO, IDECyL). `rasterize.py`/`score.parcels`/`live.py` read `config.*` dynamically so the engine can retarget the grid per region.

**`europe` now covers the whole continent** (widened from the Variscan-belt window `(-10,36,16,55)` to `(-10.5,35,30,68)` — Iberia to the Baltics/Poland, Britain/Ireland to the Balkans, and the Fennoscandian Shield). This surfaced a real bug in `_egdi_wfs` (used by `egdi_granite`/`egdi_faults`/`egdi_felsic_volcanics`): **a single GetFeature request over a large bbox comes back geographically thinned server-side**, proportional to query area — confirmed empirically by probing the exact same 10°×10° box (containing Cornwall's real, INSPIRE-mapped granite) at shrinking tile sizes: 10° → 532 polys/0 in Cornwall, 5° → 746/0, 3° → 964/24, 2° → 969/25 — it plateaus at 3°. Pagination alone doesn't catch this (each page legitimately reports "no more results" well short of the true total — confirmed the *pagination* itself is correct via a `resultType=hits`-adjacent exhaustion test), so `_egdi_wfs` now auto-tiles any bbox wider/taller than `_EGDI_TILE_DEG` (3.0°) into sub-tiles and concatenates (each tile independently cached by `net.http_get`, so a rebuild replays cheaply). This took the whole-continent granite fetch from 269 → 11,458 polygons and correctly recovered Cornwall's classic Sn-W granite. Hotspot tuning for the wider region: `target_pctl=94.0`/`top_k=22` empirically gives the best geographic spread (22/22 slots filled — Iberia/Sweden/Czechia/France all distinctly represented; higher percentiles merge more into the #1 Iberia blob while dropping smaller real clusters below `min_cells` instead of shrinking them). `cell_size` bumped 5→7km (honest given 1:1M source + ~3x area). One real geological finding this surfaced: **Bergslagen (Sweden) and Cornwall initially showed identical rock-bottom posterior values** — Bergslagen genuinely is skarn/VMS-hosted (not granite-related, correctly scored low), while Cornwall's absence was the tiling bug; after the fix Cornwall's real granite-hosted tin recovers and Sweden's Fennoscandian pegmatite belt (dominated by W, not Sn/Li) surfaces as hotspot #2. Each region's descriptive subtitle now comes from a single source of truth (`Region.label` → `commodities.json.region_description` → `meta.json` → `Sidebar.subtitleFor`) instead of hardcoded per-region-key strings in the frontend, which had gone stale twice already when a region's scope changed.

**Iberian Pyrite Belt (`iberia` region) — a DIFFERENT mineral system (VMS base metals), same engine:** base metals (Cu/Zn/Pb/Ag) are dominantly volcanogenic massive sulfide, hosted by **felsic volcanics, not granite**, so they can't ride the granite model. The Region now carries its own commodity sets (`commodities`/`hotspot_commodities`/`proxy_commodities`, defaulting to the granite globals), and `iberia` overrides them with `config.VMS_COMMODITIES` = Cu/Zn/Pb/Ag + swaps the evidence: `granite=live.egdi_felsic_volcanics` (EGDI 1:1M lithology filtered to finegrainedigneous/pyroclastic/rhyolitoid — the IPB Volcano-Sedimentary Complex), `occurrences=live.egdi_basemetals` (EGDI split into `_EGDI_GRANITE`/`_EGDI_VMS` so granite regions don't fetch base metals). Validated: WofE gives **positive weights for the volcanic host** (in-volcanics W+ 1.44, near-volcanics W+ 1.22) and ~neutral faults — the VMS signature (stratabound, not vein) — and the hotspots land squarely on the belt (Riotinto/Huelva, Aljustrel/Neves-Corvo, Tharsis). `pipeline.run`/`export` read `region.commodities` (not `config.COMMODITIES`) and glob `prospectivity_*.tif` per region so each region's assets carry only its own metals; the UI subtitle/legend/step-text derive their metal names from `commodity_models` (no hardcoded "Sn·W·Li"). `iberia` hotspots drill to Spanish parcels via `live.cadastre_spain` (Catastro INSPIRE CP WFS `ovc.catastro.meh.es/INSPIRE/wfsCP.aspx`, `cp:CadastralParcel`, urn:4326 lat,lon bbox — GDAL flips; id `nationalCadastralReference`, area `areaValue` m²; no cadastral value/municipio in INSPIRE CP) now wired into `_COUNTRY_CADASTRE` — so the Cu/Zn/Pb/Ag buy-list works over the IPB, and the europe-view Spanish hotspots drill too. The WFS is slow (~1 min/box), so `iberia` uses a modest drill (`drill_radius_km=4`, `parcel_min_area_ha=2`); net.http_get caches the responses. **Remaining gaps:** Portugal has no open bbox cadastre (its IPB hotspots stay raster-only); Spanish parcels have no price (Catastro CP carries none, no DVF-equivalent), so the base-metal buy-list has no budget affordability yet.

**Spain** uses IGME 1:1M geology + national BDMIN; hotspots drill to Catastro parcels with legal-concession friction (IDECyL for Castilla y León via `config.CYL_PROVINCES`, else MITECO national). **Europe** uses EGDI (`geologicunitview` granite, `hike_all_faults_layer`; occurrences from the full INSPIRE inventory `egdi_mineraloccurrences_inspire` filtered server-side via an OGC `fes:Filter` to commodity ∈ {Tin, Tungsten, Lithium, **Tantalum, Niobium**} — the CRM subset layer had almost no tin, this gives ~1.3k Sn / 1.4k W / 120 Li / 80 Ta / 66 Nb). Europe hotspots **drill to per-country open cadastres** via a **border-aware** bbox provider (`live.parcels_for_bbox` → uses `live.natural_earth_countries()` to find every country the hotspot bbox overlaps and concatenates each one's cadastre: France IGN Géoplateforme WFS `cadastre_france`, Czechia ČÚZK INSPIRE WFS `cadastre_czechia`, Germany/Saxony GeoSN ALKIS-WFS `cadastre_saxony`; so a hotspot straddling the DE/CZ Erzgebirge pulls both. Portugal's DGT cadastre is too sparse and Spain-in-EU-view uses its own municipio drill, so those stay raster-only). The `Region` carries either a `parcels_provider` (Spain, municipio-resolved) or a `parcels_bbox_provider` (Europe, a `drill_radius_km` box around each hotspot's representative point); `pipeline.drill_hotspot` branches on which is set. Hotspots correctly surface Iberia/Erzgebirge/Massif Central/Cornwall. **EGDI gotchas:** JSON output 500s (fetch GML); WFS 1.1.0 + EPSG:4326 returns lat,lon (`_egdi_wfs` flips to lon,lat; `egdi_occurrences` builds points from the `longitude`/`latitude` columns to sidestep it); `cql_filter` is ignored (use `fes:Filter`, no bbox param alongside it — clip client-side). Catastro ATOM parsers need `r.encoding='utf-8'` or ñ/í mangle. Data caveat: EGDI 1:1M granite is coarse.

**Two front-ends:** `subsuelo/web/index.html` is the original self-contained vanilla-JS UI (served by `run_ui.py` on :8000, the reference implementation). `subsuelo/webapp/` is a **Vite + React + TypeScript + react-leaflet** port (`npm --prefix subsuelo/webapp run dev` on :5175, launch config `subsuelo-react`) — same guided UX, componentized (App / Sidebar / MapView / Legend / Intro), data shapes typed in `src/types.ts` (`npm run typecheck`), reading region data from `public/regions` (symlinked to `out/web/regions`). See `webapp/README.md`. Both consume the same `out/web/regions/<key>/` assets.

**Multi-region UI:** `export.py` mirrors each run's web assets into `out/web/regions/<key>/` and maintains `out/web/regions.json`, so building several regions (`run_region.py spain` → export; `run_region.py europe` → export) leaves all of them available. The UI reads `?region=<key>` (defaults to the first in `regions.json`), loads that region's folder, and shows an **Area** dropdown to switch (full reload per switch). It has two modes (tab-toggle): **"Find land by budget"** (the default) and **"Explore the map"**. Explore is the guided 1-2-3 flow (Prospectivity → Prospective areas → Land parcels) with a first-load intro, an **on-map legend**, region-adaptive stats, a metal selector that swaps the heatmap **and filters the occurrence dots**, and fetches each clicked hotspot's `hotspot_parcels_<rank>.geojson` on demand (hotspots with `n_parcels=0`, e.g. European ones, just zoom).

**Budget/intent search funnel** (`webapp/src/components/SearchPanel.tsx` + `data.searchParcels`): the "Find land by budget" mode is the product's spine — state a target metal + region + total budget + min size + exclude-claimed, get a **ranked cross-region buy-list** of acquirable parcels. It reads a flat per-region `parcels_index.json` (emitted by `export._build_parcel_index`: one row per drilled parcel with area/€ha/score/claimed/centroid + **per-metal `prosp_<C>`** sampled from each metal's raster at the parcel centroid, since a hotspot's `top_commodity` is mostly Sn — the per-metal posterior gives the true "Li ground"). Ranking by a per-metal posterior is **relative** (posteriors are tiny absolute probabilities, ~0.07 max — keep above-median for that metal, normalize 0–1 for display); total-budget mode walks the ranked list marking parcels affordable until spent ("€500k buys ~N ha"). Picking a result (`App.onPickParcel` + `pendingNav`) switches region → drills the hotspot → focuses the parcel, so a buy-list spans both regions. Real €/ha exists only for France (`no price` tag elsewhere; a `priced only` toggle restricts).

Outputs are written to `./out/`: `prospectivity.tif`, `parcels_scored.geojson`, `top_parcels.csv`, `demo_map.png` (demo only), and `occurrences.geojson` (live only). `run_ui.py` additionally writes `out/web/` (reprojected GeoJSON, colorized raster PNG, `meta.json`, `index.html`) and serves it at http://127.0.0.1:8000.

## Web UI

- `subsuelo/web/export.py` — reprojects `out/` artifacts to EPSG:4326 (Leaflet needs lat/lon), warps the raster to a colorized PNG overlay (alpha ramp: faint low → opaque high), writes `meta.json`, and emits `parcels_index.json` (the flat buy-list index for the budget search, with per-metal `prosp_<C>` sampled at each parcel centroid). **Masks sea cells to transparent** using Natural Earth 50m land (`live.natural_earth_land`, cached at `ingest/_ne_land_50m.geojson`) — otherwise distance-to-granite/fault is nonzero over water near coastal features and the heatmap bleeds offshore.
- `subsuelo/web/index.html` — self-contained Leaflet SPA: prospectivity heatmap + parcel-score choropleth + (live) BDMIN mineral-occurrence points colored by commodity, ranked-parcel sidebar, minerals panel, and a **target-commodity selector** (All/Sn/W/Li) that swaps the heatmap + re-ranks parcels per metal. Loads Leaflet + CARTO tiles from CDNs (needs internet).

**Per-commodity models:** the pipeline trains a separate WofE posterior for each of `config.COMMODITIES` = Sn/W/Li/**Ta**/**Nb**/**U**/**Mo**/**Be**/**Bi**/**Ba**/**As**/**Sb** — all granite-related metals that share the same evidence (near-granite/near-fault/in-granite), so each extra one is near-free. U/Mo/Be/Bi are clean granite/greisen/pegmatite fits; **`config.PROXY_COMMODITIES` = {Ba, As, Sb}** are vein/fault hydrothermal — the granite model is only a proxy, so their `commodity_models.reliable` is forced false (UI shows "indicative — granite model is only a proxy", regardless of sample size). The base metals (Cu/Zn/Pb/Ag) are NOT in this set — they're a different mineral system (VMS), handled by the separate **`iberia` region** (see below). The original set was Sn/W/Li/Ta/Nb (same evidence layers, deposits = showings whose `sustancia`/EGDI `commodity` mentions that metal), writing `prospectivity_<C>.tif` and per-parcel `score_<C>`/`prospectivity_<C>` columns. Ta/Nb are pegmatite by-products so they ride the same granite/pegmatite evidence "for free"; they're selectable per-metal layers but **`config.HOTSPOT_COMMODITIES` = Sn/W/Li** still drives hotspot detection (Ta/Nb shouldn't redraw where the clusters are). EGDI filters occurrences server-side by exact commodity (Europe: Ta 80 / Nb 66 reliable); IGME classifies from the Spanish `sustancia` string with a Li→W→Sn→Ta→Nb priority (first match wins), but the per-metal training re-scans `sustancia` so Ta stays reliable in Spain (~42) even though most Sn-Ta showings take the Sn primary tag. `config`-level `MIN_TRAIN` (8) flags statistically thin models (Li has only 2 showings in the Cáceres window → flagged "indicative"). Getting a reliable Li model needs a larger window (national scope) with more Li occurrences. Scoring precomputes parcel→cell membership once (`score.parcels.cell_membership`) so N rasters cost ~1× the zonal loop.
- `run_ui.py` chains pipeline → export → static server; `--live` uses `run_live` instead of `run_demo`, `--no-run` reuses existing `out/`.

## Architecture

This is a Python geospatial pipeline for mineral prospectivity screening and land acquisition scoring in Extremadura, Spain (Sn-W-Li belt). The project lives in `subsuelo/subsuelo/` (inner package).

**Pipeline flow** (`run_demo.py` orchestrates):
1. **Ingest** (`ingest/`) — generate or fetch evidence layers (geology contacts, faults, geochemistry, parcels, price listings)
2. **Model** (`model/wofe.py`) — binarize evidence layers and run Weights of Evidence to produce a posterior probability raster
3. **Score** (`score/parcels.py`) — compute per-parcel composite score via zonal stats

**Spatial context:** EPSG:25829 (ETRS89/UTM29N — the correct zone for Cáceres; was 25830/zone-30N, which put the window ~400 km east in the sea), 100×100 km window around Cáceres, 500 m cell resolution.

**Scoring formula** (from `config.py`):
```
score = 0.5 · prospectivity_rank + 0.3 · cheapness + 0.2 · (1 − friction)
```

**Key files:**
- `subsuelo/config.py` — central config: `BBOX`, `CRS`, `CELL_SIZE`, `SCORE_WEIGHTS`, `ENDPOINTS`
- `subsuelo/model/wofe.py` — Agterberg/Bonham-Carter WofE with 0.5 continuity correction
- `subsuelo/ingest/synthetic.py` — synthetic data generator (used by `run_demo.py`)
- `subsuelo/ingest/live.py` — live ingestors, **validated 2026-07 against real services** (IGME ArcGIS REST + Catastro INSPIRE)
- `subsuelo/ingest/rasterize.py` — turns live vector geology into the model's distance/mask grids (needs `scipy`)

## Status of live data ingestors

`ingest/live.py` is validated and driven by `run_live.py`:
- **IGME (ArcGIS REST, not WFS)** — pinned services: `BasesDatos/IGME_BDMIN_Indicios` (mineral occurrences, layer 1), `Cartografia_Geologica/IGME_GeologicoCaceres_200` (layer 5 = contacts+faults via `TIPO`, layer 7 = lithology via `DLO`). Queried with `MapServer/<id>/query?f=geojson`, paged by `resultOffset` (maxRecordCount 1000). Occurrence commodity is classified from the Spanish `Sustancia` string (see `config.TARGET_COMMODITIES`, Li-first priority); granite host rock from `config.GRANITE_LITHOLOGIES`.
- **Catastro INSPIRE ATOM** — per-municipio parcel GML zips at `.../CadastralParcels/{prov}/{code}-{NAME}/A.ES.SDGC.CP.{code}.zip`; the URL NAME must match the province ATOM feed exactly. `config.CATASTRO_MUNICIPIOS` defaults to four mineralized Cáceres municipios. INSPIRE CP has no cadastral value, so `cadastral_eur_ha` is NaN. Requires Python `requests`+`certifi` (the Spanish gov TLS cert doesn't chain in curl's default bundle).
- **Catastro Minero legal concessions** (`live.mining_concessions`) — the REAL legal mining-rights registry from the **MITECO national WFS** (`geoportal.minetur.gob.es/cgi-bin/mapservcm`, typename `derechosMineros`): concession POLYGONS with `nombre` + `leyenda` (type/status → granted `…O` / pending `…T` / lapsed `…CD`). GeoJSON output is broken server-side, so fetch **GML 3.1.1** and parse with GDAL; geometry is lon/lat despite the SRS label (force `set_crs(4326)`). In `run_live`, granted+pending concessions are the `claimed` friction layer (real polygon overlap — ~24% of Cáceres parcels fall in one). This replaced the earlier workings-buffer proxy.
- **BDMIN Explotaciones** (`live.mining_workings`) — real mine workings/exploitations (points, active/abandoned); kept as a context layer. (Watch out: IGME's `Catastro_Minero_DOM` service is the **Dominican Republic**, not Spain — use the MITECO WFS above for legal rights.)
- **Per-country EU cadastres** — `cadastre_france` (IGN Géoplateforme WFS, id `idu`), `cadastre_czechia` (ČÚZK INSPIRE `cp:CadastralParcel`, GML), `cadastre_saxony` (GeoSN ALKIS-WFS `geodienste.sachsen.de/aaa/public_alkis/vereinf/wfs`, typename `ave:Flurstueck`, native EPSG:25833, id `flstkennz` / area `flaeche` m² / municipio `gemeinde` — Germany's cadastre is per-Bundesland, this is Saxony only, covering the Erzgebirge). Dispatched by `live.parcels_for_bbox` which is **border-aware** (fetches every country the hotspot bbox overlaps). Gotcha: Saxony's `wfs_geosn_inspire-cp/guest` paths 403 to a CMS page; the working bbox WFS is the AdV-simplified `aaa/public_alkis/vereinf/wfs`.
- **France DVF land prices** (`live.dvf_commune_eur_ha`) — real French land-transaction prices from geo-DVF per-department CSVs (`files.data.gouv.fr/geo-dvf/latest/csv/{year}/departements/{dep}.csv.gz`). Filters to bare-land sales (`nature_mutation=='Vente'` & `type_local` NaN & `surface_terrain>1000`), takes the median €/ha per commune (clipped 200–80k, cached per dep). `cadastre_france` joins them onto parcels by `code_insee` → `cadastral_eur_ha`, so French hotspot parcels get a **real cheapness** component (Spain still has none — INSPIRE CP carries no value and idealista needs OAuth).
- **Eurostat estimated land prices** (`live.eurostat_land_price` + `live.nuts_price_polygons`) — where no real transaction price exists (Spain, Czechia; France keeps DVF), parcels get an *estimated* €/ha from Eurostat `apri_lprc` (arable land, latest year) joined by NUTS2 region (GISCO NUTS2 boundaries, NUTS2→NUTS1→NUTS0 fallback, clipped 500–80k). `export._fill_estimated_prices` sjoins each un-priced parcel centroid to its NUTS region → `eur_ha` + `price_kind='est'` (real DVF = `'real'`, none = `'na'`); the UI tags estimates "est." and the pick carries the estimate onto the ParcelDetail card. So the budget funnel now works over Spain (incl. IPB — Extremadura ≈€9.9k/ha, Andalucía ≈€21k) and Czechia (≈€15k), not just France. Germany reports no arable-land price to Eurostat, so those NUTS are filled from **`config.DE_LAND_PRICE_EUR_HA`** (Destatis "Kaufwerte für landwirtschaftliche Grundstücke" 2023, per Bundesland/NUTS1: Sachsen €15,606, Sachsen-Anhalt €23,033) merged into the NUTS lookup — so Saxony parcels are priced too (static figures, refresh annually). Net: every drilled area now has a real or estimated €/ha (only a handful of border-edge parcels stay `'na'`).
- **idealista API** — still needs approved OAuth credentials; a token would give real (not estimated) Spanish transaction prices.
- **Natura 2000 (MITECO WFS)** — attempted for friction; falls back to empty (friction 0) if the WFS is unreachable.

Friction = `clip(protected_frac + claimed_frac, 0, 1)`. The UI cache-busts data/raster fetches (`?_=<ts>`) since pipeline outputs change between runs.
